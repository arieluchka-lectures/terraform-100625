n8n runs in port 5678

ec2 type used - t3.micro

remember to attach igw to vpc and route table and to igw on 0.0.0.0 and attach this route table to the subnet

Start-Service ssh-agent
ssh-add +{link to key}
ssh -A -i {link to key} ec2-user@{ec2-eip}   (try using ssh -i /path/to/new-key.pem -o IdentitiesOnly=yes ec2-user@{ec2-eip} if too many attempts)

n8n commands to download image and run container:
2) sudo systemctl start docker
3) sudo docker volume create n8n_data
sudo docker run -it -d --rm --name n8n -p 5678:5678 -e N8N_LISTEN_ADDRESS=0.0.0.0/0 -v n8n_data:/home/node/.n8n docker.n8n.io/n8nio/n8n
(both lines together in 1 cmd)

in EC2 - install docker:
(in user_data/script.sh)  #!/bin/bash
sudo update yum -y
1) sudo yum install docker -y


updated command (with all tcp allowed): (removed -it --rm as well)
4) sudo docker run -d --name n8n -p 5678:5678 -e N8N_HOST=0.0.0.0 -e N8N_PORT=5678 -e N8N_PROTOCOL=http -e N8N_SECURE_COOKIE=false -v n8n_data:/home/node/.n8n docker.n8n.io/n8nio/n8n
